## Getting Started

### Prerequisites

- Node.js 18+ installed
- npm or yarn package manager
- Firebase project (for authentication)

### Installation

1. Clone the repository:
```bash
git clone https://github.com/shivaganesht/matrixo-website-deployment.git
cd matrixo-website-deployment
```

2. Install dependencies:
```bash
npm install
```

3. Set up environment variables (create `.env.local`):
```env
# Firebase Configuration
NEXT_PUBLIC_FIREBASE_API_KEY=your_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_auth_domain
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your_storage_bucket
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
NEXT_PUBLIC_FIREBASE_APP_ID=your_app_id

# Resend (Email)
RESEND_API_KEY=your_resend_api_key

# Site URL
NEXT_PUBLIC_SITE_URL=https://matrixo.in
```

4. Run the development server:
```bash
npm run dev
```

5. Open [http://localhost:3000](http://localhost:3000) in your browser

## Security & Compliance

### Security Hardening
- **Input Validation**: Server-side validation on all API routes
- **HTML Sanitization**: DOMPurify for safe content rendering
- **Firestore Security Rules**: Row-level security, owner-only operations
- **File Upload Controls**: MIME type validation, size limits
- **Password Security**: Firebase secure password reset, email verification
- **Rate Limiting**: Protection against brute force and DoS
- **Encryption**: crypto-js for sensitive data at rest
- **HTTPS Only**: All communications encrypted
- **CORS Configuration**: Restricted cross-origin requests
- **Cache Control**: No-store headers on admin/sensitive routes

### Authentication Flow Security
1. Email signup → verification email → sign out → user verifies → sign in
2. Google OAuth: popup (production), redirect (beta, cross-domain)
3. GitHub OAuth: popup modal
4. Password reset: Firebase `sendPasswordResetEmail`

### Firestore Security Rules Pattern
```
UserProfiles: public read, owner-only write, no delete
Usernames: public read, auth create, owner update, no delete
applications: public create, auth read/update, no delete
vibecode_registrations: auth create/read, no update/delete (immutable)
skillDNA_users: owner-only (uid match on all operations)
Location data: public read, NO write
```

### Storage Rules Pattern
```
Resumes: auth read/write (PDF only, 5MB max)
Profile photos: public read, owner write (images only, 5MB max)
Event images: public read, auth write (images only, 10MB max)
Default: DENY ALL
```
